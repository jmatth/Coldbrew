Welcome To Coldbrew

Congratulations, if you're seeing this page in your browser it means you've successfully gotten Coldbrew to compile and run! See below for details on how to get started adding your own code.

Adding Routes

All the code that handles HTTP requests lives in src/main/java/Coldbrew/App.java. Look there to start adding your own code.

What About IDE Support?

You can import the project into Intellij or Eclipse. Be sure to import it as a Gradle project so the IDE picks up the dependencies from build.gradle.

I Want To Style Things

Coldbrew comes with the Bootstrap framework for styles, and you can put your custom styles in src/main/scss/styles.scss.

What About Javascript?

Looks like the js build pipeline is broken, tweet @jmatth92, open a Github issue, or track down Josh Matthews if you're at a hackathon and think he's around.

What's With All The {% verbatim %}{% tags %}{% endverbatim %} In The HTML Files?

Those are part of Pebble, the default template engine for Coldbrew. You can use a different engine by changing the code in App.java (and of course updating the templates to use the new format). See here for a list of template engines supported by Spark.

I Need To Do Something You Didn't Include

In the likely event that you need to connect to a database, use an API client, or use external libraries in general, you should add those dependencies to build.gradle. After that you should be able to import and use them in your code.
